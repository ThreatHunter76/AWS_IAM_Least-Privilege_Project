# AWS IAM Least Privilege Project

## Overview
This project demonstrates how an AWS IAM misconfiguration can create excessive access risk and how to remediate it using the Principle of Least Privilege.

## Objectives
- Create an IAM user with misconfigured permissions
- Use AWS CLI to test privileged actions
- Fix the configuration using least privilege
- Document the vulnerability and remediation

## Tools Used
- AWS IAM
- AWS CLI
- VS Code
- Windows Command Prompt

## Project Workflow
1. Environment setup
2. IAM user creation
3. Insecure policy attachment
4. Access key creation
5. AWS CLI identity verification
6. Removal of insecure policy
7. Attachment of secure least-privilege policy
8. Validation of restricted access

## Key Security Concept
The Principle of Least Privilege means users should only have the minimum permissions required to perform their tasks.
